# I18N Translate 项目说明

面向 IntelliJ 平台的插件：**在编辑器中维护多语言消息文件（properties / JSON / YAML），并对接云翻译 API 批量写入目标语言文件**。本文梳理仓库结构、配置模型与主业务流程，便于阅读代码与二次开发。

---

## 1. 项目概述

| 项 | 说明 |
|----|------|
| **插件 ID** | `com.github.xucux.I18nTranslate` |
| **构建产物** | Gradle `intellij` 插件，`sinceBuild=231`，目标 IDE 约 2023.1～2026.1（见 `build.gradle.kts`） |
| **语言** | Kotlin + JDK 17 |
| **核心能力** | 从代码选中内容或从源消息文件选中 key，写入源文件并翻译到多个目标文件；支持整文件按 key 顺序翻译 |
| **翻译引擎** | 阿里云机器翻译（HTTP ROA）、腾讯云 TMT（TC3-HMAC-SHA256）、DeepL |

云厂商 **未使用官方 Java SDK**，改为 `java.net.http` + 自实现签名，减少传递依赖体积（见 `build.gradle.kts` 注释）。

## 3. 架构分层与数据流

### 3.1 分层关系

1. **表现层**：`action/*`、`view/*` — 收集用户输入、校验、弹窗，最终在后台任务中调用 `I18nTranslateWorkflow`。
2. **编排层**：`I18nTranslateWorkflow` — 读取 `GlobalPluginConfigService` / `ProjectI18nConfigService`，选择字符集与写入模式，通过 `TranslationExecutors` 获取执行器，调用 `translateWithStats` 后由 `FlatMessageFileIo` 在 `WriteAction` 中写文件。
3. **翻译抽象**：`TranslationExecutor`（`translation/api`）— 对上层只暴露 `translate(text, source, target) -> TranslateOutcome`。
4. **引擎实现**：`AliyunTranslationExecutor`、`TencentTranslationExecutor`、`DeepLTranslationExecutor` — 各自调用 `remote` 包下 `*ApiClient`。
5. **持久化与配置**：`GlobalPluginState` / `ProjectI18nState` + `PluginStoragePaths` — properties 文件落盘。

### 3.2 一次「译到多目标」的简化数据流

```
Action / Dialog
    → I18nTranslateWorkflow.translateToTargets(...)
        → GlobalPluginConfigService.getState()  // 引擎、编码、写入模式、凭证、腾讯地域等
        → TranslationExecutors.create(state)     // 具体 Executor
        → 对每个 TargetEntry：
              TranslationExecutors.translateWithStats(executor, stats, ...)
              → FlatMessageFileIo.upsertAfterAnchor(...)  // 保持与源文件 key 相对顺序
        → I18nStatsService 累计调用与计费字段（成功/失败/字数或字符数）
```

## 4. 配置模型

### 4.1 存储位置

| 范围 | 路径 | 内容 |
|------|------|------|
| **全局** | `%USER_HOME%/i18n_translate_plugin/general.properties` | UI 语言、翻译引擎、文件编码、写入模式、各云凭证、腾讯云地域等 |
| **全局统计** | 同目录 `stats.properties` | 调用次数、成功/失败、累计翻译量（由 `I18nStatsService` 维护） |
| **项目** | `<project>/.idea/i18n-translate-project.properties` | 源消息文件路径、源语种、多个目标（路径 + `SupportedLanguage`） |

### 4.2 关键类型

- **`GlobalPluginState`**：与 `general.properties` 读写对应；含 `TranslateEngine`、`FileWriteMode`、`tencentRegion` 等。
- **`ProjectI18nState`** / **`TargetEntry`**：项目级源文件与目标列表。
- **`TencentTranslateRegion`**：腾讯云 `X-TC-Region` 可选值枚举；在 **腾讯云凭证弹窗**（`EngineCredentialsDialog`）中选择，持久化为 `tencent.region`。
- **`GlobalPluginConfigService`**（Application）、**`ProjectI18nConfigService`**（Project）：内存态 + `applyState` / `reloadFromDisk`。

设置界面：**`I18nTranslateConfigurable`**（`plugin.xml` 注册在 `other` 下），与上述服务同步；引擎凭证（含腾讯云 SecretId/SecretKey 与 **地域**）通过 **`EngineCredentialsDialog`** 写回 `globalSnapshot` 再随 Apply 落盘。

## 5. 文件与格式

- **`MessageFileKind`**：按扩展名识别 `properties` / `json` / `yaml|yml`。
- **`FlatMessageFileIo`**：统一读写为「有序扁平 Map」；支持在末尾追加/更新 key、按「锚点 key」在目标文件中插入以保持顺序（与源文件 key 顺序对齐）。
- **字符集**：由全局配置 `charsetName` + `CharsetUtil` 解析；可选自定义编码开关。

---

## 6. 翻译引擎与远程调用

| 引擎 | 执行器 | HTTP 客户端 | 要点 |
|------|--------|-------------|------|
| 阿里云 | `AliyunTranslationExecutor` | `AliyunMtApiClient` | `https://mt.cn-hangzhou.aliyuncs.com/api/translate/web/general`，JSON Body + ROA 签名 |
| 阿里云签名 | — | `AliyunRoaSignature`（`util`） | MD5(Base64 body)、HMAC-SHA1、`Authorization: acs ...` |
| 腾讯云 | `TencentTranslationExecutor` | `TencentTmtTc3ApiClient` | `tmt.tencentcloudapi.com`，TC3-HMAC-SHA256，`X-TC-Region` 来自配置 |
| DeepL | `DeepLTranslationExecutor` | `DeepLApiClient` | 官方 REST，`DeepL-Auth-Key` |

公共 HTTP：**`PluginHttpApiClient`**（JDK `HttpClient` 单例封装）。

限流：**`FixedWindowRateLimiter`**（阿里云约 50/s、腾讯约 5/s，在执行器侧使用）。

工厂：**`TranslationExecutors`** — `create(GlobalPluginState)` 分支创建对应 Executor，并统一在 `translateWithStats` 中写统计。

## 7. 领域与语言映射

**`SupportedLanguage`**：插件内部枚举；为各云 API 提供 `aliyunCode`、`tencentCode` 等（见 `domain/SupportedLanguage.kt`）。新增语种时需同步各引擎文档中的 code。

---

## 8. 编辑器与 Action

- **`I18nEditorPopupGroup`**：挂在 `EditorPopupMenu` / `ProjectViewPopupMenu`；按上下文动态组合子项：
  - 非 i18n 文件 + 有选中文本：添加 key / 添加并翻译到全部或指定目标；
  - 源 i18n 文件 + 检测到 key + 已配置目标：`TranslateToTargetDynamicGroup`、`TranslateKeyToAllTargets`；
  - 源 i18n 文件：`TranslateEntireSourceFileAction`（整文件 key）。
- **`I18nKeyDetector`**：在 properties/json/yaml 中根据光标解析当前 key。
- **`I18nPaths`**：判断是否认定为 i18n 文件、是否为当前项目配置的源文件。
- **`I18nActionSupport`**：BGT 线程、选中文本等公共逻辑。

具体 Action 类名见 `plugin.xml` 的 `<actions>`。

## 12. 小结

插件以 **「全局配置 + 项目路径」** 驱动，**`I18nTranslateWorkflow`** 串联 **读源 → 云翻译 → 按锚点写目标**，**`TranslationExecutors`** 隔离多引擎差异；持久化均为 **UTF-8 Properties**，路径集中在 **`PluginStoragePaths`**，便于排查与备份。
